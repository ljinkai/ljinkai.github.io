Indie Maker 一个人的创业

Mar 17 2019
扫描二维码

我们从大学毕业，到踏入职场，然后结婚生子，过着日子。这个过程，我相信每个人不管在这里面的哪个阶段，有一个共同的点，就是一直想“赚钱”。一开始满怀信心，觉得自己无所不能，“可能”赚很多的钱，因为年轻，可以不断的试错。慢慢的你会更清楚的认清自己，理想和现实之间有巨大的差距。作为一个工作敲代码10多年的我来说，试错的机会成本变大，有一些最近的思考和心得和大家分享。

数字游民

我是一名程序员，一直尝试做些产品来当业余项目来做，希望能有些收入，也做过一些尝试，写我的博客、做程序员站立办公桌、开发小工具、知乎上做付费课程，虽然也有些收入，却一直不得要领，也没有明确的目标。我一直在反思和实践着，什么是我们一直在寻找的路径呢，有没有一些人的成功经验，前一段时间看到的一个油管上的视频，给了我很大的启发。

Pieter Levels是一个Nomad, 一位数字游民，一台电脑、有Wifi就可以工作的人，自己一个人做的所有产品，每个月都有几万美金的收入，全世界各地一边旅游，一边做着自己想做的事，目前在巴厘岛，大家可以关注他的Twitter。

💡 Idea -> 🛠 Build -> 🚀 Launch -> 🌱 Grow -> 💰 Monetize -> 🤖 Automate -> 🚪 Exit

他总结了自己的一套方法论，共7个步骤，包括了整个产品的生命周期。从想法、构建、运行、增长、变现、自动化、退出，这7个关键步骤讲开。作为一个数字游民，尝试一年12个月，做12个产品，每个月发布一版，从一开始的每月$100、到$300、到$700到现在的每月$20000美金的收入。视频的最后答疑环节，有人问他：“如果自己的技术不怎么样，我该怎么开始呢？” 他说：“你不会的可以Google，我就是这样的，我一开始对技术一窍不通, 不懂的就去Google, 真的，我是认真的”。

仔细想想，我们要有全产品思维，技术人员有的时候过分强调我要用多新多牛的技术，这就有些狭隘了，我们要更多的从全产品来考虑你做的产品，为用户真正解决什么问题，只有创造了价值你的代码才有价值，舍本求末，结果就是自嗨却没有用户买账。

下面就引出了我们的重点: Indie Maker, 这跟我们国内的独立开发者的概念比较接近，Indie Maker是独立、创作、自由，通过自己独立的做产品来完成变现，也就是一个人的创业。国外这种理念和人非常多，ProductHunt上面每天都有新Indie Maker做的产品。

这些Makers

Arthur， 法国人，一个数字游民，创建了一个数字游民的社区，鼓励大家用30天完成自己的项目，一步步实现自己的小的商业模型，体验编程和数字游民的生活方式。https://nomadhouse.io/


Dong Nguyen, 越南人，Indie Game Maker, 因为制作手机游戏Flappy Bird而闻名，专注做像素类的小游戏。 http://www.dotgears.com/


Corey, 美国人，10多年的产品经验，致力于把想法变成现实，做过很多款产品。https://www.coreygwin.com/


Ethan， 澳大利亚人，年龄很小，Indie Maker 通过做web相关的在线产品变现，他最近正在做一个邮件看板的产品，已经上线。https://ethan.link/


上面列举的4名Indie Maker，来自世界各地。他们通过技术来将自己的想法变成现实，并通过为人们提供有价值的服务，来实现盈利，他们不是每天去上班，而是以自己喜欢的生活方式来完成自己的产品，在自己喜欢的地方，在自己喜欢的环境，而不是坐在那些办公大楼的格子间里。这些人有一个共同的特点，就是面向小众的市场，帮助用户解决问题，来实现盈利。因为小而专注，也就能切分出特定的市场，实现某个领域的独特产品。

他们也在用上面Pieter Levels所说的7个步骤来实践自己的产品迭代，通过这套方法，作为我们来说，是不是也可以做一个Indie Maker，做出自己的产品呢？

Talk is cheap. Show me the code

你可以看看你的周围，百分之九十的人都把自己停留在了”想法”阶段，想法很多，却一直不能执行构建。究其原因，是你的认知没有想明白你做的事情，这也是人和人的差别，看似是自己的懒惰，其实是自己的思考力不够，一旦你想明白自己要做的事，和能解决的问题，带来的价值，你会自己给自己想办法来前进，而不是靠外力来推进。

首先自己要建立全产品思维的概念，打破自己仅仅会一项技术的小富即安的心理。从想法、开发、推广、运营、盈利等等这些你都需要自己搞定，不会的就去学，本身学习能力也是一个人的竞争优势，短时间内快速的掌握一项技能也是需要不断的强化和练习的，现在互联网上的教程、视频那么多资源，要学会去利用。

快速迭代出产品并上线和用户建立连接。越拖越耗费你的精力，你需要快速的产出阶段性产品，和用户建立连接，让用户给予你反馈，然后不断的迭代版本。这样做的目的是纠偏，防止你做的产品和用户真实需求之间相悖，越早的和用户建立反馈机制，就能更好的促进产品走在正确的方向上。

找到你的同类。一个人很孤独，但是一群人却能互相带来温暖和鼓励，如果你有过减肥失败或者其他半途而废的经历，你就会明白我说的，我们都是人，都有惰性，如果是一群理念相同的伙伴在一起，你看到他们都那么努力和每天的进步，你就会同样的充满能量。一群人在一起也会互相的交流出更多的资源，激发更多的灵感，本身这样一群人就会形成一个网，让彼此的知识得以互相传递。如果你发现跟你周围的人好像说不到一块，又有自己独特想法，那么请坚持下去，找到你的同类。

因为要长期的践行Indie Maker，我会建立一个微信群，大家可以分享交流关于Indie Maker的内容。我也会第一时间分享相关的内容、技术、工具等等，希望在这个群里能遇到更多的同类。为了能保证群的质量，不欢迎那些非Indie Maker者的旁观，设定此群为收费群，欢迎那些认同Indie Maker理念的小伙伴。
