如何从日常发现中挖掘赚钱产品？

Nov 20 2020
扫描二维码

我们每天都会在工作和生活中遇到问题，然后解决问题。在这个过程中，抱有一颗好奇心，往往会有新奇的发现。

很多人对于很多事情只停留在了表面，如果在接触事物的过程中，带着一些产品挖掘的心态，往往就能发现有意思的切入点，从而打造自己的赚钱产品。

下面我们就以实操的心态来分享下我最近的3个发现案例，如果在这过程中你能有什么所得，那么纯属偶然，哈哈。

1、FFCreator

作为一个程序员而言，每天早上我们都有浏览资讯的习惯，我在看Github上开源代码库趋势的时候，发现了一个有意思的开源库。

FFCreator - 一个轻量的nodejs短视频加工库，高效稳定、极速合成，人人都能视频制作

这个是大约2周前才刚刚开源出来的库，被Star的才4百多，由腾讯新闻前端团队出品。也就是说这个库被人知道的还不多，技术稳定性还是不错的，毕竟是大厂开源的东西。

他的官网的介绍说明了这个技术的应用场景：

您只需要添加几张图片或视频片段再加一段背景音乐，就可以快速生成一个很酷的视频短片。
今天，短视频已成为一种越来越流行的媒体传播形式。像微视和抖音这种app，每天都会生产成千上万个精彩短视频，而这些视频也为产品带来了巨大的流量。 随之而来，如何让用户可以快速生产一个短视频；或者产品平台如何利用已有的图片、视频、音乐素材批量合成大量视频就成为一个技术难点。
FFCreator是一种轻量又简单的解决方案，只需要很少的依赖和较低的机器配置就可以快速开始工作。它基于node.js开发, 普通前端工程师既可以轻松上手。 并且它模拟实现了animate.css90%的动画效果，您可以轻松地把 web 页面端的动画效果转为视频。

是的，现在短视频很火，如果把已有的图片、视频、音乐素材批量合成就成了一个市场需求，想想如果是做知识、资讯类短视频，利用现在网上的海量的文字、图片、视频，稍作加工，那么你一天产生的视频就是别人可能一年的量。

就在这个开源代码发布的几天后，我收到了来自知乎的一条通知。

是不是觉得有点那个味了，我们知道短视频是巨大的流量池，如果我们能做个工具帮助人们方便的制作出短视频，将海量的图文转换成短视频，而且有现成的技术开源库，知乎也才刚开始这方面的尝试，打造一个赚钱的产品「Don’t Say So Much」

2、iOS的Shadowrocket下载

之前一直用Android手机，最近有一天，工作中需要给iOS手机安装翻墙软件，才了解到iOS手机上安装Shadowrocket(俗称小火箭🚀)是需要美区apple id， 而是是要付费购买价格$2.99。

其实翻墙的需求一直就有，尤其对于那些不太懂技术的人，墙是一个难以逾越的障碍，而且仅是各种fq软件的名字就让人傻傻分不清楚，更别说各种的下载安装等等操作，所以很多人宁愿花点钱解决问题，告诉他一个方便的解决方案，所以需求端的刚需很强烈。下面我们看看我付费的流程是怎样的。

首先我搜索关键字”ios安装shadowrocket”排名第一的是github上的一个开源库。其实这里面就有一个SEO的逻辑，因为github上的网站权重很高，所以利用github来做一些关键字流量的入口是非常适合的，比自己搭网站要高明很多。

然后进去之后，这里面其实就是一个readme.md文档，并附上链接，最终都是引导到了自己的网站

我尝试了如下的方式：

1、免费账号共享，试了好几个，都不能使用，基本上就是死胡同

2、Apple ID账号方式，需要翻墙下操作，对于本身就需要翻墙的同学来说，这个门槛就很高3、最后花了58块钱在线购买美区Apple ID并在线安装的方式，体验最方便，其实这也是被“推荐”的方式

点击购买后，填写下信息后，支持支付宝、微信付款，付款完成后，即收到账号信息。

是的，我的iOS安装Shadowrocket的问题是解决了。但是我们要保持一颗好奇心，再来多挖掘下。

整个过程都是自动化完成的，可以说只要有apple id其实我们自己也完全可以打造一个完全类似的产品，因为人家已经证明这个刚需是能获取到付费用户的。而注册美区apple id的成本是多少呢？

相信细心的同学肯定能找到解决方案，我用的58元付费的方式的成本是一个美区apple id + $2.99, 那这个中间的利润就能算出来，我甚至觉得都不要自己建网站，github上建个开源项目，留下一个微信号，加微信->付款->发账号「Don’t Say So Much」

3、前端面试题

做前端开发的同学肯定经常上掘金这个网站，如果你把全部的热榜排序，你会发现一个很有意思的事情。

是的，排名靠前的都跟一个主题有关“面试题”，这就是实实在在的刚需，程序员对这方面的需求已经用点赞来投票了。

那我们再来看看Github上的情况，大家已经用Star来投票了，竟然会有1~2万的Star。写到这，不由得感叹一句，Github是宝藏啊。

看到这妥妥的刚需，如果好好整理下“面试题”的内容，就是一个长尾的流量。

如果围绕面试题这个点，就可以打造一个产品出来。每年都有大量的毕业生进入这个行业，而每年都有很多人去跳槽找新工作，对于那些刚毕业几年的同学来说，面试是真的，面试题也是真的要刷的。微信小程序里已经有人在做这样的事情了，面试题带来流量，通过高阶课程来收费，这就是一个很好的赚钱模式。「Don’t Say So Much」

以上就是要和大家分享的案例，其实打造一个能赚钱的产品并不是很难，首先需求我们找到刚需，所谓刚需，就是真的有用户愿意付费购买。一旦确定刚需，剩下的就是要想法设法的实现它。

一个赚钱产品的打造最重要的是思考层面，技术次之。我们要多动脑筋，仔细观察，发掘那些可能的切入点，保持一颗好奇心，往下多挖一层往往会有惊喜，在日常发现里就有很多的宝藏。
